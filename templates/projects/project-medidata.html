Medidata: Electronic Health Record (EHR) Functional Prototype

Back
Medidata is an Electronic Health Record (EHR) functional prototype designed strategically to simplify patient medical data, thereby, allowing doctors to more quickly review patient history prior to visits.

PROJECT DETAILS
Need: Hospitals are rapidly adopting Electronic Health Records (EHR) systems. Currently, these systems have an overwhelmingly amount of data stored per patient. Via market research, it was discovered that doctors are spending on average 10-15 minutes reading this data before meeting with patients.
Objective: Develop an interface that increases doctor's productivity using the EHR, thereby reducing patient wait times.
Approach: Designed a web application with four main modules (generic patient information header, ICD9 wheel, timeline of patientâ€™s previous visits, and clinical note session) that utilize a RESTful API to pull data from a mock patient database. The information was structured using: search-centric design, sequential ordering, expandable views, color labeling, and graphical views.
Project Duration: 6 months; Spring 2015
Group Members: Joanne Bruno, Derek Burgman, Matthew Snyder, Michael Draughon
My Role(s): UI UX Lead, Front-End Developer, Project Manager, Scribe
Final Result: Created an interactive EHR web application as a functional prototype to better visualize patient data through the use of a relevancy filterable timeline and by mapping patient history over an ICD9 Tree.
